# PLANS.md

Use this format for implementation plans handed to an implementation agent or engineer.

## Plan Template

### Goal

What are we implementing?

### Scope

What is included?

### Out of Scope

What should not be implemented?

### Relevant Docs

Which ADRs/requirements should be read first?

### Files / Areas to Modify

List expected folders and files.

### Implementation Steps

1. Step one
2. Step two
3. Step three

### Validation Commands

List commands to run.

### Risks / Open Questions

Capture any uncertainties.

### Completion Checklist

- [ ] Code implemented
- [ ] Tests added/updated
- [ ] Docs updated if needed
- [ ] Checks pass
